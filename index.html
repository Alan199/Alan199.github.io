写博客的心酸历程

Veröffentlicht am 2017-04-27

第一篇博客，献给我写博客的心酸历程

以下是大佬分享的写博客教程:

$ hexo new “文件名”

写markdown需要用到一些语法：

可以参考Cmd Markdown简明语法手册

插入图片时遇到了小麻烦。百度了一下，有两种方法可以插入本地图片：

第一种，上传本地的：

CodeFalling/hexo-asset-image的插件来加载本地图片：

1.首先确认_config.yml 中有 post asset folder :true。
Hexo 提供了一种更方便管理 Asset 的设定：post_asset_folder
当设置post_asset_folder为true参数后，在建立文件时，Hexo
会自动建立一个与文章同名的文件夹，可以把与该文章相关的所有资源都放到那个文件夹，如此一来，便可以更方便的使用资源。

2.在hexo的目录下执行npm install https: //github.com/CodeFalling/hexo-asset-image –save（需要等待一段时间）。

3.完成安装后用hexo新建文章的时候会发现_posts目录下面会多出一个和文章名字一样的文件夹。图片就可以放在文件夹下面。

第二种，远程地址：

申请一个七牛云账号，把图片存储到云，然后复制外链就可以得到地址

或者先上传到github仓库再去调用，使用远程地址

然后又遇到了图片太大不整洁的问题：

图片大小解决方法：

我使用七牛云存储图片，所以通过七牛云图片样式改变大小：
先建立图片样式，然后在外链中加样式接口
例如(http://ooupgqdn7.bkt.clouddn.com/QQ%E5%9B%BE%E7%89%8720170425215512.png-450300)
-450300是样式接口，手动加上去

最后再分享一款写markdown的软件:typora
感谢小旋旋的推荐，这款软件可以直接预览效果，且页面简洁，比用word直接敲好用了不知多少倍
